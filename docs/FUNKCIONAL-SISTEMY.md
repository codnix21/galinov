# Функционал CRM «Галинов» (текущее состояние)

Платформа недвижимости на Laravel 12: каталог объявлений, модерация, документы, договоры, CRM риэлтора, админ-панель.

**Роли в системе:** `admin`, `realtor`, `client`, `guest` (справочник `roli`).

**Сотрудники** = администратор + риэлтор (`isStaff()`).

---

## 1. Администратор (`admin`)

Доступ: `/admin/*` (только при роли admin, иначе 403).

### 1.1. Дашборд (`/admin`)

- Счётчики: пользователи, объявления (всего / активные / на модерации / продано), договоры, риэлторы, клиенты.
- Ленты: последние объявления, пользователи, договоры.

### 1.2. Пользователи (`/admin/users`)

| Действие | Описание |
|----------|----------|
| Список | Пагинация 20, **поиск** по ФИО, email, телефону |
| Создание | ФИО, email, пароль, роль, телефон |
| Редактирование | Профиль, роль, блокировка |
| Удаление | Удаление учётки (нельзя удалить себя) |
| Блок / разблок | Флаг `zablokirovan` — middleware `check.blocked` не пускает на сайт |

### 1.3. Объявления (`/admin/properties`)

**Фильтры и поиск** (класс `PropertyCatalogFilter::applyAdminList`):

| Параметр | Назначение |
|----------|------------|
| `search` | Название, описание, адрес, город |
| `type` | `apartment`, `house`, `commercial`, `land` |
| `operation` | `sale`, `rent` |
| `status` | `draft`, `active`, `pending_review`, `sold`, `rented`, `inactive` |
| `city_id` | Город из справочника |
| `min_price` / `max_price` | Цена |
| `sort` | `newest`, `price_asc`, `price_desc`, `area_desc` |

**Действия:** полный CRUD, назначение владельца (риэлтор/админ), статус, до 10 фото, параметры дома, **собственники с долями**, геоточка (DaData).

### 1.4. Договоры (`/admin/contracts`)

| Действие | Описание |
|----------|----------|
| Список | **Поиск** по названию объекта, ФИО/email сторон, примечаниям |
| Создание / редактирование | Объект, стороны, риэлтор, статус, даты, цена, примечания |
| Удаление | — |
| PDF | Экспорт договора |

### 1.5. Отчёты (`/admin/reports`)

**Фильтры периода и срезов:**

| Параметр | Назначение |
|----------|------------|
| `date_from` / `date_to` | Период (по умолчанию неделя) |
| `filter_property_status` | Статус объявления |
| `filter_contract_status` | Статус договора |
| `filter_property_tip` | Тип недвижимости |
| `filter_user_role` | Роль пользователя |
| `sort_top` | Топ пользователей: `properties_count`, `familia`, `email` |
| `sort_dir` | `asc` / `desc` |

**Экспорт:** PDF, CSV, XLSX.

### 1.6. Журнал изменений (`/admin/audit-logs`)

| Фильтр | Назначение |
|--------|------------|
| `search` | ФИО, email, действие, ID объекта, текст |
| `deystvie` | Код действия (создание, изменение, договор…) |
| `obyekt` | `property`, `contract`, `user` |
| `date_from` / `date_to` | Даты |

Отображение: действие на русском, объект со ссылкой, таблица «поле / было / стало».

### 1.7. База данных (`/admin/database`)

- Создание резервной копии SQL.
- Скачивание бэкапа.
- Восстановление из файла.

### 1.8. Справочники (`/admin/dictionaries`)

- **Города:** добавление, удаление.
- **Статусы объявлений:** код + название.
- **Статусы договоров:** код + название.

### 1.9. Импорт объявлений (`/admin/import`)

- Скачать **шаблон CSV** (заголовки на русском: Название, Цена, Город…).
- Импорт CSV/XLSX: обязательные «Название» и «Цена»; опционально тип, операция, статус, email владельца, описание.

### 1.10. Массовые операции

- Смена статуса у выбранных объявлений (`/admin/bulk/properties`).
- Смена статуса у выбранных договоров (`/admin/bulk/contracts`).

### 1.11. Также доступно админу (как сотруднику)

- Вся **модерация** (см. раздел 4).
- **Рабочее место риэлтора** и CRM (админ видит все объекты/клиентов, не только свои).
- **Обучение** риэлторов (`/training`).
- Личный кабинет, профиль, договоры, уведомления — как у авторизованного пользователя.

## 2. Риэлтор (`realtor`)

Доступ к CRM: middleware `realtor.training` (риэлтор + админ).

### 2.1. Рабочее место (`/realtor`)

- Статистика: объявления (всего, активные, модерация, черновики, продано, сдано), договоры, очередь модерации (чужие на проверке).
- CRM: клиенты, открытые задачи, предстоящие показы, подборки.
- Ближайшие задачи и показы.
- Договоры, ожидающие подтверждения риэлтора.

### 2.2. Портфель объявлений (`/realtor/properties`)

**Фильтры** (`PropertyCatalogFilter::applyRealtorPortfolio`):

| Параметр | Назначение |
|----------|------------|
| `status` | Статус объявления |
| `search` | Название, адрес |
| `type` | Тип |
| `operation` | Продажа / аренда |

Только объявления, где риэлтор — владелец (`polzovatel_id`).

### 2.3. CRM — клиенты (`/realtor/clients`)

| Фильтр | Значения |
|--------|----------|
| `status` | `new`, `in_progress`, `deal`, `lost` |
| `search` | ФИО / email клиента |

- Привязка клиента из базы (роль client).
- Карточка клиента: контакты, заметки, договоры, показы, задачи.
- Поиск клиентов для привязки (API).

### 2.4. CRM — задачи (`/realtor/tasks`)

| Фильтр | Назначение |
|--------|------------|
| `filter` | `open` (по умолчанию), `done` |

- Типы: звонок, встреча, показ, другое.
- Срок, клиент, объект, отметка выполнения.

### 2.5. CRM — показы (`/realtor/showings`)

| Фильтр | Назначение |
|--------|------------|
| `filter` | Предстоящие / прошедшие (`past`) |

- Выбор клиента и **объекта из списка активных** (не ID вручную).
- Дата/время, результат (заинтересован, не подошло, не пришёл, к сделке).

### 2.6. CRM — подборки (`/realtor/collections`)

- Создание подборки, публичная ссылка `/podborka/{token}`.
- Добавление объектов из каталога или из избранного.
- Удаление объектов / подборки.

### 2.7. Входящие заявки

| Раздел | URL | Действия |
|--------|-----|----------|
| Заявки на объект | `/realtor/inquiries` | Список, отметка «в обработке» |
| Заявки на подбор | `/realtor/selection-requests` | Список, обработка |
| Запросы информации | `/realtor/info-requests` | Ответ, закрытие |

Типы запросов информации: срок владения, документы, обременения, согласие супругов, другое.

### 2.8. Обучение (`/training`)

- Материалы для риэлторов.
- PDF по темам: холодные/горячие клиенты, работа с объявлениями.

### 2.9. Объявления и сделки (как у клиента + расширения)

- Создание / редактирование / черновики / документы / публикация на модерацию.
- **Собственники** с долями (отдельная форма сохранения).
- **Экспресс-сделка** по объекту: выбор покупателя, автозаполнение договора.
- Создание договоров вручную (поиск объекта, клиента, риэлтора через API).
- Подтверждение договоров, УКЭП, загрузка скана, PDF, аренда — печатная форма.
- Модерация чужих объявлений и документов (см. раздел 4).

## 3. Клиент (`client`)

Обычный пользователь: покупатель / собственник.

### 3.1. Личный кабинет (`/cabinet`)

- Список **своих объявлений** (пагинация).
- Счётчики: всего, активные, на модерации, продано.
- Счётчики договоров (где участник).
- Блок **«Следующий шаг»** (Lean): документы, модерация, подтверждение договора.

### 3.2. Объявления

| Действие | Описание |
|----------|----------|
| Создать | Тип, операция, цена, город (DaData), адрес, фото, параметры дома |
| Черновики | `/properties/drafts` |
| Редактировать | Все поля + статус + фото |
| Документы | Чек-лист по типу объекта и сделке, загрузка файлов и реквизитов, проверка ЕГРН по кадастру |
| Собственники | Доли 100%, основной продавец (если есть право управления) |
| На модерацию | Кнопка после готовности документов |
| Удалить | Свои объявления |

**Статусы объявления:** черновик → (документы) → на модерации → в каталоге (active) → продано / сдано / неактивно.

**Степпер публикации** на карточке: Черновик → Документы → Модерация → В каталоге.

### 3.3. Покупка и сделки

| Функция | Описание |
|---------|----------|
| Заявка на объект | Имя, телефон, email, комментарий (на активном объявлении) |
| Запрос информации | Типовые вопросы по активному объявлению |
| Избранное | Добавить / убрать, список `/favorites` |
| Онлайн-покупка | «Купить без риэлтора» → договор → оплата (Robokassa или тест) |
| Договоры | Список своих сделок, просмотр, согласование, УКЭП |
| Полный отчёт | `/properties/{id}/report` — документы, заявки, история (для владельца) |

Ручное создание договора из меню **недоступно** — редирект с подсказкой использовать покупку на карточке.

### 3.4. Заявка на подбор (`/selection-request`)

- Форма с контактами и **сохранёнными фильтрами** каталога (город, цена, тип…).
- Можно отправить из каталога без отдельной формы (гость/авторизованный).

### 3.5. Профиль (`/profile`)

- **Информация профиля:** ФИО, телефон, пол, email, биография, аватар.
- **Персональные данные:** паспорт, ИНН, СНИЛС (для договоров и модерации).
- **Смена пароля.**
- **Telegram:** привязка уведомлений.
- **Документы продавца:** паспорт в профиле (используется в объявлениях).
- **Удаление аккаунта.**

### 3.6. Уведомления (`/notifications`)

- Список, счётчик непрочитанных, «прочитать все».

## 4. Модерация (админ + риэлтор)

URL: `/moderation` (middleware `staff.moderate`).

### 4.1. Очередь объявлений

| Фильтр | Назначение |
|--------|------------|
| `q` | Название, адрес, ФИО/email владельца и риэлтора |
| `docs` | `all`, `ready` (паспорт+ЕГРН проверены), `not_ready` |
| `sort` | `newest`, `price`, `client` |
| `dir` | `asc` / `desc` |

- **Одобрить** → статус `active` (в каталоге).
- **Отклонить** → черновик + причина (`prichina_otkaza_mod`), уведомление владельцу.

### 4.2. Модерация документов (`/moderation/documents`)

- Очередь файлов на проверку.
- Подтвердить / отклонить / отправить на повторную проверку.
- Просмотр реквизитов и файлов.

## 6. Общий функционал системы

### 6.1. Каталог и поиск (`/properties`)

Показываются только объявления со статусом **active**.

**Фильтры:**

| Параметр | Описание |
|----------|----------|
| `search` | Умный поиск: `#123` / `№123` по ID; иначе слова в названии, описании, адресе, городе |
| `type` | Квартира, дом, коммерция, участок |
| `operation` | Продажа / аренда |
| `city_id` | Город |
| `min_price` / `max_price` | Цена |
| `min_rooms` / `max_rooms` | Комнаты |
| `min_area` / `max_area` | Площадь, м² |
| `min_floor` / `max_floor` | Этаж |
| `has_photos` | Только с фото |
| `sort` | Новые, цена ↑↓, площадь ↓ |

**Фильтры для домов** (при `type=house` или активных фильтрах дома):

`tip_doma`, `est_tsokol`, `garazh`, `parking`, `internet`, `min_ploshchad_uchastka`, `max_ploshchad_uchastka`.

**Карта:** те же фильтры, метки по `geo_shirota` / `geo_dolgota` (Яндекс.Карты).

**Похожие объявления:** если по фильтрам пусто — до 6 похожих активных.

### 6.2. Карточка объявления

- Галерея, описание, характеристики, карта/панорама (Яндекс).
- Собственники (таблица долей) — публично на карточке.
- Избранное (для вошедших).
- Степпер публикации (для владельца/управляющего).
- Заявка, запросы информации, онлайн-покупка, экспресс-сделка (по правам).
- Журнал изменений (владелец / админ).
- История версий статуса.
- Ссылки: ипотека, Росреестр, полный отчёт.

### 6.3. Документы на объект

Зависят от **типа** и **операции** (продажа / аренда):

| Тип | Продажа (пример) | Аренда (пример) |
|-----|------------------|-----------------|
| Квартира | Паспорт, ЕГРН, право собственности | Паспорт, ЕГРН, договор аренды |
| Дом | + кадастровый план | + кадастр, договор аренды |
| Участок | ЕГРН земли, право | ЕГРН земли, аренда |
| Коммерция | + ЕГРЮЛ | + ЕГРЮЛ, аренда |

Статусы документа: не загружен, на проверке, проверен, отклонён.  
Паспорт может браться из **профиля** (verified).  
Проверка ЕГРН по кадастровому номеру (внешний реестр / JSON).

### 6.4. Собственники (`sobstvenniki_nedvizhimosti`)

- Несколько владельцев, доли в сумме 100%.
- Один **основной** продавец.
- Копирование всех собственников в договор как продавцы.

### 6.5. Договоры и сделки

**Статусы договора:** черновик, ожидает, активен, завершён, отменён.

**Стороны:** владелец (`vladelets_id`), покупатель (`pokupatel_id`), риэлтор (`rieltor_id`).

**Согласование** (зависит от того, кто создал):

- Создал риэлтор → подтверждают владелец и покупатель.
- Создал клиент → подтверждает вторая сторона + риэлтор.
- Онлайн-покупка → автозаполнение, оплата, УКЭП.

**Дополнительно:**

- Подпись УКЭП (имитация/сервис).
- PDF и печатная форма аренды.
- Загрузка скана (риэлтор/админ).
- После завершения продажи/аренды — смена статуса объявления (sold/rented).

**Список договоров** (`/contracts`): поиск `q` (ID, объект, ФИО), сортировка `newest` / `price` / `date_start` / `id`.

### 6.6. Оплата

- **Robokassa:** Result / Success / Fail URL.
- **Тестовый шлюз** (если включён в конфиге).
- Страница оплаты по договору онлайн-покупки.

### 6.7. Уведомления

- В приложении (`/notifications`).
- Telegram (webhook, привязка в профиле).
- События: модерация, заявки, договоры, покупка и др. (`AppNotifier`).

### 6.8. Интеграции

| Сервис | Назначение |
|--------|------------|
| DaData | Подсказки города и адреса (только auth) |
| Яндекс.Геокодер / Карты | Карта объявления, панорама |
| Росреестр / кадастр | Ссылка на карту, проверка ЕГРН |
| Robokassa | Оплата |

### 6.9. Журналирование и версии

- **Журнал изменений** (`zhurnal_izmeneniy`): объявления, договоры, пользователи — кто, что, когда; таблица diff.
- **Версии статусов** (`versii_statusov`): история смены статуса объявления/договора.
- **Аудит** в админке — тот же журнал с фильтрами.

### 6.10. Безопасность и качество контента

- Блокировка пользователей.
- Фильтр мата в названии/описании объявлений (`TextCensor`).
- Проверка этажа ≤ этажности дома.
- CSRF, авторизация по ролям, 403/404 на чужие объекты.
- Политика просмотра: черновики и модерация скрыты из каталога; отчёт и история — по правам.

### 6.11. Lean-подсказки

На дашбордах и в кабинете — цепочка ценности и кнопки «следующий шаг» (модерация, документы, договоры, CRM).
