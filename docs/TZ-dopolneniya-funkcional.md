# Дополнения к ТЗ «Агентство недвижимости»

Документ для обновления раздела **4.1 «Функциональные требования»** в файлах `Техническое задание.docx` / `.pdf`.

**Дата сверки с кодом:** май 2026  
**Стек реализации:** Laravel 12, PHP 8.4, MySQL, Vite, Blade.

---

## 1. Что уже описано в текущем ТЗ (кратко)

| Роль | В ТЗ сейчас |
|------|-------------|
| **Гость** | Главная, описание, регистрация, вход |
| **Клиент** | ЛК (ФИО, email, телефон, пароль, выход), поиск/фильтры, избранное, свои объявления (добавить/удалить) |
| **Риэлтор** | Всё у клиента + поиск по названию/стоимости, модерация объявлений, изменение данных |
| **Администратор** | Всё у риэлтора + управление объявлениями, договоры, отчёты, пользователи (блокировка, роли, фильтры), «редактирование категорий» (формулировка неясна) |

**Вывод:** в ТЗ отражена только базовая витрина и CRUD. **Большая часть реализованного функционала в ТЗ не описана.**

### 2.1. Роль «Гость» (дополнить)

**Каталог и поиск**

- Просмотр каталога объявлений с фильтрами: тип недвижимости (квартира, дом, коммерция, участок), операция (продажа/аренда), город, цена, комнаты, площадь, этаж, сортировка (дата, цена, площадь).
- Текстовый поиск по словам, адресу, описанию и номеру объявления (формат `#123`).
- Просмотр объявлений на интерактивной карте (Яндекс.Карты) с учётом фильтров каталога.
- Просмотр карточки опубликованного объявления: фото, характеристики, описание, карта, контакты автора (для активных объявлений).
- Просмотр **полного отчёта по объекту** (паспорт объекта, статус документов, сводка по сделкам — без доступа к чужим договорам).

**Информационные страницы**

- Страница «О нас / контакты».
- Страница «Помощь» (инструкции по работе с системой).
- Страница «Схема процессов» (Lean: этапы от заявки до закрытия сделки).
- **Ипотечный калькулятор**: расчёт платежа, переплаты, графика; привязка к цене объявления по параметру URL.

**Заявки и подборки**

- Отправка **заявки на объект** с карточки объявления (имя, телефон, email, комментарий) без обязательной регистрации.
- Просмотр **публичной подборки** риэлтора по ссылке-токену (`/podborka/{token}`).

**Платежи (системные URL)**

- Обработка callback платёжной системы Robokassa (result/success/fail) — для завершения онлайн-оплаты после авторизации покупателя.

*Примечание для ТЗ:* отдельный пункт меню «Поиск» убран — поиск объединён с каталогом «Объявления».

### 2.2. Роль «Клиент» (дополнить)

**Личный кабинет**

- Сводный **личный кабинет**: статистика объявлений и сделок, подсказки «следующий шаг» (Lean workflow).
- **Уведомления** в интерфейсе (колокольчик): список, счётчик непрочитанных, отметка прочитанным, переход по ссылке из уведомления.
- Опциональные уведомления в **Telegram** при привязке аккаунта к боту.

**Профиль и документы продавца**

- Редактирование профиля: ФИО, email, телефон, биография, **аватар**.
- **Персональные данные** (зашифрованное хранение): серия/номер паспорта, дата и орган выдачи, ИНН, СНИЛС — с валидацией форматов.
- Раздел **«Документы продавца»** (пошагово):
  1. Заполнение данных паспорта;
  2. Загрузка скана паспорта на проверку;
  3. Опционально — ИНН/СНИЛС;
  4. Ссылка на документы по каждому объявлению (ЕГРН, право собственности и т.д.).
- Индикатор **«Готовность к размещению»** при проверенном паспорте в профиле (учитывается в чек-листе объявления).

**Объявления (владелец)**

- Создание объявления: тип, операция, цена, город (справочник), адрес (подсказки **DaData**), кадастровый номер, площадь, комнаты, этаж, описание, **несколько фотографий**, геокоординаты (автогеокодирование).
- Режимы размещения: **от клиента**, **риэлтор за клиента**, **риэлтор от агентства** (поля автора размещения).
- Черновики: список, редактирование, удаление.
- **Степпер публикации** на карточке: черновик → документы → модерация → опубликовано.
- **Чек-лист документов на объект** (последовательная загрузка): паспорт (может засчитываться из профиля), ЕГРН, право собственности и др. в зависимости от типа объекта и сделки; проверка кадастрового номера, ссылка на карту Росреестра.
- Отправка объявления **на модерацию** после выполнения чек-листа.
- Просмотр причины отклонения модератором и повторная отправка.
- **Журнал изменений** по своему объявлению (кто и что менял).

**Покупка и сделки (клиент — покупатель/арендатор)**

- **Онлайн-покупка** с карточки активного объявления: создание договора, шаг оплаты (**Robokassa** и/или тестовый шлюз в демо-режиме), страница завершения.
- **Экспресс-сделка** (договор с автозаполнением) — для владельца объявления или по сценарию с риэлтором; выбор второй стороны по ФИО.
- Просмотр **своих договоров** (где клиент — владелец, покупатель или указанный риэлтор).
- **Подтверждение / отклонение** договора в статусе «ожидает» (многостороннее согласование: владелец, покупатель, риэлтор — по правилам, кто создал договор).
- **Подписание договора УКЭП** (имитация квалифицированной подписи) со стороны покупателя.
- Скачивание **PDF договора**, печать бланка аренды, завершение аренды при активном договоре.
- *Не для клиента:* ручное создание договора через форму «Новый договор» — только для риэлтора/админа; клиент оформляет сделку через «Купить онлайн» / «Экспресс-сделку».

**Прочее**

- **Избранное**: добавление/удаление активных объявлений.
- Подтверждение email (стандартный механизм Laravel Breeze).
- Подтверждение пароля для чувствительных операций.

### 2.3. Роль «Риэлтор» (дополнить)

**Обучение**

- Раздел **«Обучение»**: материалы для риэлтора, PDF по темам (холодные звонки, горячие лиды, работа с объявлениями).

**CRM (рабочее место)**

- **Дашборд риэлтора**: сводка по клиентам, задачам, показам, модерации, договорам.
- **Клиенты CRM**: список, карточка, статусы (новый, в работе, сделка, потерян), заметки, поиск клиентов.
- **Задачи**: создание, срок, привязка к клиенту, отметка выполнения, удаление.
- **Показы**: планирование показа объекта клиенту, перенос, отмена, заметки.
- **Подборки объектов**: создание подборки для клиента, добавление/удаление объектов, **публичная ссылка** для клиента, создание подборки из избранного.
- **Заявки на объекты** (inbox): просмотр заявок с карточек объявлений, отметка «обработано».

**Модерация (сотрудник)**

- Очередь объявлений **на модерации**: просмотр, **одобрение** (публикация в каталог), **отклонение** с указанием причины (нельзя модерировать **собственное** объявление).
- Очередь **документов на проверку** (паспорт, ЕГРН и т.д.): подтверждение, отклонение с комментарием, повторная автопроверка.

**Договоры (риэлтор)**

- **Создание договора** вручную: выбор объекта, владельца, покупателя, риэлтора (поиск по ФИО/email), цена, даты, примечания.
- Просмотр **всех договоров** в системе (в отличие от клиента — только свои).
- Согласование договоров: подтверждение/отклонение, очередь «на подтверждение».
- **Загрузка скана** подписанного договора (для аренды после активации).
- **Экспресс-сделка** от имени риэлтора с привязкой покупателя к CRM-клиенту.

**Объявления**

- Все функции клиента по своим объявлениям + просмотр портфеля **«Мои объявления (риэлтор)»** с фильтрами.

**Интеграции**

- Привязка **Telegram** для уведомлений (генерация deep-link на бота).

### 2.4. Роль «Администратор» (дополнить)

**Админ-панель (`/admin`)**

- **Дашборд**: ключевые показатели, быстрые ссылки.
- **Пользователи**: список, поиск/фильтры (имя, email, телефон, дата регистрации), создание, редактирование, удаление, **блокировка/разблокировка**, смена роли (кроме смены собственной роли).
- **Объявления**: полный CRUD без ограничений владельца (создание, редактирование, удаление любого объявления).
- **Договоры**: полный CRUD, экспорт PDF.
- **Отчёты агентства**: статистика за период (объявления, договоры, пользователи), **экспорт PDF, CSV, XLSX**.
- **Журнал аудита** (`audit-logs`): поиск по изменениям в системе (объявления, договоры и др.).

**Права администратора**

- Полный доступ к модерации, CRM, обучению, договорам (как у риэлтора).
- Просмотр полного отчёта по объекту и журнала изменений для любого объявления.
- Участие в сделках и оплате (обход ограничений покупателя при необходимости в демо/сопровождении).

*Уточнить в ТЗ:* пункт «Редактирование категорий» заменить на **«Справочники: города, статусы объявлений и договоров»** (реализованы через БД и сидеры, отдельного UI категорий нет) **или** описать планируемый модуль справочников, если будет добавлен.

## 4. Расхождения: что в ТЗ написано неточно или не реализовано

Исправить или сформулировать как «планируется», чтобы не было противоречий на защите.

| Пункт ТЗ | Фактическая реализация | Рекомендация для ТЗ |
|----------|------------------------|---------------------|
| «Фильтрация товаров по категориям» | Фильтры по **типу недвижимости** и операции, не «товары» | Заменить формулировку на «тип объекта, продажа/аренда, город, цена…» |
| «Удаление добавленных своих объявлений» | Клиент **удаляет свои объявления**; «добавленные» звучит как избранное | Разделить: избранное / свои объявления / черновики |
| «Подтверждение или отклонение договоров» (только админ) | Подтверждают **стороны сделки и риэлтор** по правилам; админ — полный доступ | Описать многостороннее согласование и роль админа |
| JWT, срок сессии 30 мин | **Сессии Laravel** (cookie), не JWT | Привести ТЗ к фактической схеме или указать «планируется JWT» |
| 2FA (TOTP) | **Не реализовано** | Убрать из обязательных или вынести в «перспективу» |
| «Редактирование категорий» (админ) | Отдельного UI нет; города и статусы — в БД | Уточнить или описать админ-справочники |
| Отдельный раздел «Умный поиск» | Поиск **в каталоге объявлений** | Не дублировать в ТЗ отдельный модуль |
| Клиент создаёт договор вручную | **Отключено** — только онлайн-покупка и экспресс-сделка | Явно указать в ТЗ сценарии для клиента |

## 5. Нефункциональные требования — что уточнить

| Тема | Предложение для ТЗ |
|------|-------------------|
| Шифрование ПДн | Персональные данные и паспорт в БД — **encrypted** (Laravel casts) |
| Пароли | Bcrypt — **соответствует** |
| Валидация | Серверная + HTML5 (maxlength, pattern) на формах паспорта, ИНН, СНИЛС |
| Производительность 100 RPS | Оставить как целевой показатель; для диплома можно сослаться на `tools/load_test.php` |
| Docker | В проекте есть `docker-compose` — добавить в «Установка и развёртывание» |
