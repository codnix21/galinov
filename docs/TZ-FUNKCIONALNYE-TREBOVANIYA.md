## 4 Требования к приложению

### 4.1 Функциональные требования

Ниже приведены функциональные требования к веб-приложению CRM для сопровождения сделок с недвижимостью (каталог объявлений, модерация, документы, договоры, CRM риэлтора, администрирование). Роли пользователей: «Гость», «Клиент», «Риэлтор», «Администратор».

---

#### 1. Функции роли «Гость»

- Просмотр главной страницы;
- Просмотр информационных страниц:
  - описание компании и контактов;
  - справка / помощь;
  - описание процесса сделки;
  - калькулятор ипотеки;
- Просмотр каталога объявлений о недвижимости:
  - отображение только опубликованных (активных) объявлений;
  - просмотр карточки объявления (описание, фотографии, характеристики, карта);
- Поиск и фильтрация объявлений в каталоге:
  - поиск по тексту (название, описание, адрес, город; по номеру объявления);
  - фильтрация по типу недвижимости (квартира, дом, коммерция, участок);
  - фильтрация по операции (продажа, аренда);
  - фильтрация по городу;
  - фильтрация по цене (минимальная и максимальная);
  - фильтрация по количеству комнат;
  - фильтрация по площади;
  - фильтрация по этажу;
  - фильтрация по количеству собственников (1, 2, 3 и более);
  - отбор объявлений с фотографиями;
  - фильтрация параметров дома (тип дома, цоколь, гараж, парковка, интернет, площадь участка);
  - фильтрация параметров земельного участка (категория земель, вид разрешённого использования, коммуникации);
  - фильтрация коммерческой недвижимости (класс здания, тип помещения, отдельный вход, парковка);
  - сортировка (сначала новые, цена по возрастанию/убыванию, площадь по убыванию);
- Просмотр объявлений на карте (те же условия фильтрации, что и в каталоге);
- Просмотр публичной подборки объектов по ссылке (без авторизации);
- Подача заявки на объект (имя, телефон, электронная почта, комментарий) по активному объявлению;
- Подача заявки на подбор недвижимости с указанием параметров поиска;
- Регистрация и авторизация:
  - регистрация в системе;
  - авторизация в системе;
  - восстановление пароля по электронной почте.

---

#### 2. Функции роли «Клиент»

Все функции роли «Гость», а также:

- Управление учётной записью в личном кабинете:
  - просмотр сводки по своим объявлениям и договорам;
  - изменение регистрационных данных (фамилия, имя, отчество, электронная почта, номер телефона, пол, биография, фотография профиля);
  - ведение персональных данных (паспортные данные, ИНН, СНИЛС);
  - смена пароля;
  - уведомления на email (модерация, договоры, заявки);
  - выгрузка персональных данных в ZIP (152-ФЗ);
  - удаление учётной записи;
  - выход из аккаунта;
- Сохранённые поиски:
  - сохранение набора фильтров каталога;
  - включение и отключение уведомлений о новых объявлениях;
  - удаление сохранённого поиска;
- Работа с уведомлениями:
  - просмотр списка уведомлений;
  - отметка уведомлений как прочитанных;
- Просмотр и поиск объявлений (каталог, карта, фильтры — в полном объёме, как у гостя);
- Ведение избранного:
  - добавление объявления в избранное;
  - удаление объявления из избранного;
  - просмотр списка избранных объявлений;
- Ведение своих объявлений о недвижимости:
  - создание объявления (название, описание, тип, операция, цена, город, адрес, площадь, комнаты, этаж, параметры дома, участка, коммерции, фотографии);
  - просмотр списка черновиков;
  - редактирование своих объявлений;
  - удаление своих объявлений;
  - управление фотографиями (загрузка, удаление);
  - указание собственников объекта и долей в праве (сумма долей 100 %, основной собственник);
  - загрузка и ведение документов на объект (чек-лист в зависимости от типа объекта и вида сделки);
  - проверка выписки ЕГРН по кадастровому номеру;
  - предупреждение при совпадении кадастрового номера с другим объявлением;
  - отправка объявления на модерацию;
  - просмотр статуса публикации (черновик, документы, модерация, в каталоге);
  - просмотр полного отчёта по своему объекту;
- Взаимодействие с опубликованными объявлениями других пользователей:
  - подача заявки на объект;
  - запрос дополнительной информации по объявлению (тип вопроса из справочника);
  - оформление заявки на подбор недвижимости (расширенная форма);
- Онлайн-сделка (покупка / аренда без риэлтора):
  - оформление договора по активному объявлению;
  - оплата по договору (платёжный шлюз Robokassa и/или тестовый шлюз);
  - просмотр статуса оплаты и завершения сделки;
- Ведение договоров:
  - просмотр списка договоров, в которых пользователь является стороной;
  - поиск договоров по объекту и участникам;
  - просмотр карточки договора;
  - просмотр степпера «Ход сделки»;
  - согласование (подтверждение) договора;
  - подписание договора усиленной квалифицированной электронной подписью (УКЭП);
  - формирование PDF договора (с текстами из шаблонов договоров);
  - просмотр печатной формы договора аренды;
  - график арендных платежей по активному договору аренды;
  - отметка оплаты по графику (для сторон сделки и сотрудников);
  - оставление отзыва о завершённой сделке (оценка 1–5, комментарий);
  - просмотр отзывов сторон по договору;
- Просмотр документов в профиле продавца:
  - загрузка паспорта и иных документов профиля;
  - просмотр статуса проверки документов.

#### 3. Функции роли «Риэлтор»

Все функции роли «Клиент», а также:

- Доступ к рабочему месту риэлтора:
  - просмотр сводной статистики по объявлениям, договорам, CRM;
  - аналитика за 30 дней (объявления, клиенты, заявки, договоры, динамика заявок);
  - просмотр очереди объявлений на модерации;
  - просмотр договоров, ожидающих подтверждения риэлтора;
- Поиск и фильтрация в портфеле своих объявлений:
  - поиск по названию и адресу;
  - фильтрация по статусу объявления;
  - фильтрация по типу недвижимости и операции;
- Модерация объявлений:
  - просмотр очереди объявлений со статусом «на модерации»;
  - поиск по названию, адресу, данным владельца и риэлтора;
  - фильтрация по готовности комплекта документов (готовы / не готовы);
  - сортировка (дата, цена, клиент);
  - одобрение объявления (публикация в каталоге);
  - отклонение объявления с указанием причины;
- Модерация документов:
  - просмотр очереди документов на проверку;
  - подтверждение документа;
  - отклонение документа;
  - повторная отправка документа на проверку;
- Ведение объявлений (расширенное):
  - создание и редактирование объявлений от имени агентства;
  - экспресс-сделка по объекту (выбор покупателя, автоматическое формирование договора);
- Ведение договоров (расширенное):
  - создание договора вручную (выбор объекта, покупателя, риэлтора);
  - поиск объектов, клиентов и риэлторов при создании договора;
  - подтверждение и отклонение договоров;
  - завершение договора;
  - загрузка скана подписанного договора;
  - формирование графика арендных платежей;
  - просмотр списка договоров, ожидающих действий;
- CRM — воронка клиентов:
  - отображение клиентов по статусам (новый, в работе, сделка, отказ);
  - перетаскивание карточек между колонками (drag-and-drop) с сохранением статуса;
- CRM — управление клиентами:
  - просмотр списка закреплённых клиентов;
  - фильтрация клиентов по статусу (новый, в работе, сделка, отказ);
  - поиск клиента по ФИО и электронной почте;
  - привязка клиента из базы пользователей;
  - просмотр и редактирование карточки клиента;
  - удаление привязки клиента;
- CRM — задачи:
  - создание задачи (тип: звонок, встреча, показ, другое);
  - просмотр открытых и выполненных задач;
  - отметка задачи как выполненной;
  - удаление задачи;
- CRM — показы объектов:
  - планирование показа (клиент, объект, дата и время);
  - просмотр предстоящих и прошедших показов;
  - фиксация результата показа;
  - редактирование и удаление записи о показе;
- CRM — подборки объектов:
  - создание подборки для клиента;
  - формирование подборки из избранного;
  - добавление и удаление объектов в подборке;
  - получение публичной ссылки на подборку;
  - экспорт подборки в PDF;
  - удаление подборки;
- Обработка входящих обращений:
  - просмотр и обработка заявок на объект;
  - контроль SLA ответа (подсветка просроченных заявок);
  - назначение риэлтора на заявку по объекту;
  - шаблоны ответов и копирование текста в буфер обмена;
  - просмотр и обработка заявок на подбор недвижимости;
  - назначение риэлтора на заявку на подбор;
  - просмотр запросов дополнительной информации по объявлениям;
  - ответ на запрос информации;
  - закрытие запроса информации;
- Обучение:
  - просмотр учебных материалов для риэлторов;
  - скачивание PDF по тематикам работы с клиентами и объявлениями.

#### 4. Функции роли «Администратор»

Все функции роли «Риэлтор», а также:

- Доступ к панели администратора:
  - просмотр сводной статистики по пользователям, объявлениям, договорам;
  - просмотр последних записей (объявления, пользователи, договоры);
  - мониторинг состояния системы (БД, диск, бэкапы, Robokassa);
- Настройки системы:
  - SLA заявок по объектам (часы);
  - контактный email и название агентства;
  - еженедельная рассылка отчёта на email (вкл./выкл., список получателей);
- Управление шаблонами договоров:
  - просмотр списка шаблонов (продажа / аренда);
  - редактирование разделов (введение, предмет, обязанности, заключение);
- Дубликаты кадастра:
  - просмотр групп активных и модерируемых объявлений с одинаковым кадастровым номером;
- Управление пользователями:
  - просмотр списка пользователей;
  - поиск пользователей по фамилии, имени, отчеству, электронной почте, номеру телефона;
  - создание пользователя (ФИО, email, пароль, роль, телефон);
  - редактирование данных пользователя;
  - смена роли пользователя (клиент, риэлтор, модератор, администратор);
  - выгрузка персональных данных пользователя (152-ФЗ);
  - блокировка пользователя;
  - разблокировка пользователя;
  - удаление пользователя;
- Управление объявлениями (полный объём):
  - просмотр всех объявлений в системе;
  - поиск и фильтрация (текст, тип, операция, статус, город, цена, сортировка);
  - создание объявления с назначением владельца;
  - редактирование любого объявления;
  - удаление объявления;
  - массовое изменение статуса выбранных объявлений;
  - импорт объявлений из файла CSV/XLSX (шаблон с заголовками на русском языке);
- Управление договорами (полный объём):
  - просмотр всех договоров;
  - поиск по объекту, сторонам сделки, примечаниям;
  - создание, редактирование и удаление договора;
  - массовое изменение статуса выбранных договоров;
  - формирование PDF договора;
- Формирование отчётов:
  - построение отчёта за выбранный период;
  - фильтрация по статусу объявлений, статусу договоров, типу недвижимости, роли пользователя;
  - сортировка рейтинга пользователей;
  - экспорт отчёта в PDF, CSV, XLSX;
- Журнал изменений (аудит):
  - просмотр истории действий в системе;
  - поиск по ФИО, email, действию, идентификатору объекта, тексту изменений;
  - фильтрация по типу объекта (объявление, договор, пользователь);
  - фильтрация по виду действия;
  - фильтрация по периоду;
  - отображение изменений полей (было / стало);
- Управление справочниками:
  - ведение списка городов (добавление, удаление);
  - ведение статусов объявлений;
  - ведение статусов договоров;
- Резервное копирование базы данных:
  - создание резервной копии;
  - скачивание резервной копии;
  - восстановление базы данных из файла;
- В CRM и на рабочем месте риэлтора администратор имеет доступ ко всем записям (клиенты, задачи, показы, подборки), а не только к собственным.

#### 6. Общие функции информационной системы

Функции, реализуемые на уровне системы независимо от роли (с учётом прав доступа):

- Аутентификация и разграничение доступа по ролям;
- Блокировка доступа заблокированным пользователям;
- Ведение справочников статусов объявлений, договоров и заявок (нормализованное хранение в БД, 3НФ);
- Каталог объявлений с единым набором фильтров и сортировки;
- Отображение похожих объявлений при пустом результате поиска;
- Геокодирование и отображение объекта на карте (сервис карт);
- Подсказки адреса и населённого пункта (интеграция DaData) для авторизованных пользователей;
- Проверка текстов объявлений на недопустимую лексику;
- Контроль корректности этажа относительно этажности здания;
- Журналирование изменений объявлений, договоров и пользователей;
- Ведение истории версий статусов объявлений и договоров;
- Многостороннее согласование договоров (владелец, покупатель, риэлтор) в зависимости от сценария создания;
- Учёт нескольких собственников объекта с долями при формировании договора;
- Подписание договора УКЭП всеми продавцами (доли) и покупателем;
- Уведомления пользователей (внутри приложения и на email);
- Уведомления по сохранённым поискам (планировщик);
- Приём платежей через Robokassa (callback-обработка результата оплаты);
- Публикация и раздача файлов (фотографии объявлений, документы, медиа по защищённым маршрутам);
- Редирект устаревшего URL поиска (`/search`) в каталог с сохранением параметров фильтрации;
- Отображение степпера «Статус публикации» для владельца объявления (черновик → документы → модерация → каталог);
- Формирование полного отчёта по объекту (документы, заявки, договоры, собственники — в объёме, разрешённом правами);
- Ссылка на панораму района и карту Росреестра с карточки объявления;
- Калькулятор ипотеки с подстановкой цены объекта;
- Планировщик напоминаний (договоры, аренда, задачи, показы);
- Планировщик еженедельного email-отчёта администратору.

### 4.3 Наполнение БД для проверки

Для полной проверки системы выполняется пересоздание схемы и загрузка тестовых данных сидера:

```bash
php artisan migrate:fresh --seed
```

Пароль учётных записей сидера: **Password123!**

| Роль | Email (пример) |
|------|----------------|
| Администратор | `demo.admin@agency.local` |
| Риэлтор | `demo.realtor.01@agency.local` … |
| Клиент | `demo.client.01@agency.local` … |

Дополнительно (без полного сброса БД):

```bash
php artisan db:seed --class=PlatformFeaturesDemoSeeder
```

На сервере Docker:

```bash
docker compose exec app php artisan migrate:fresh --seed --force
```
